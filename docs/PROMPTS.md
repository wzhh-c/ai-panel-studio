# 核心 Prompt 记录文档

## 1. [SDD阶段] 领域模型与数据建模

**Prompt**：
> 你是资深软件架构师。基于 PRD 摘要，输出领域模型定义，含核心实体、属性、业务规则和实体间关系。实体包括 Discussion、Expert、Transcript、Consensus、Divergence。不需要写代码。

**意图**：在动手编码前，先建立清晰的数据模型，确保后续设计与业务需求对齐。

**挑战**：AI 初始版本遗漏了 `sequence` 字段和 `color` 标识，可能影响发言顺序确定性和专家视觉区分。

**修正**：追加要求增加 `sequence` 字段和 `color` 字段，确保发言按序号排序且专家有专属颜色。

---

## 2. [SDD阶段] API 契约设计

**Prompt**：
> 你是 API 设计师。设计 RESTful API 端点（路径、方法、请求/响应 JSON）和 SSE 事件流格式，输出为 OpenAPI 3.0 YAML。包含创建讨论、列表、详情、开始、结束、重新生成阵容、SSE 流。

**意图**：锁定前后端交互契约，避免后续联调时出现接口不一致问题。

**挑战**：AI 最初将“生成阵容”和“开始讨论”合并为一个接口，导致用户无法在确认阵容后再开始。

**修正**：明确要求拆分为两个独立接口，并增加“重新生成阵容”端点。

---

## 3. [DDD阶段] 前端 UI 组件生成

**Prompt**：
> 你是资深 React 前端工程师。实现首页、阵容确认页、讨论室三个核心页面。首页展示讨论列表 + 发起新讨论；确认页展示主持人+专家阵容（含颜色、立场）；讨论室三栏布局（专家列表、Transcript 流、共识分歧），所有区域独立滚动，响应式适配超宽屏/桌面/平板/手机。

**意图**：以设计驱动开发，先构建用户可见的交互界面，再对接后端逻辑。

**挑战**：AI 生成的响应式布局在移动端折叠逻辑不完善，专家列表在窄屏下无法切换。

**修正**：补充明确的 Tailwind 断点要求和移动端 Tab 切换逻辑。

---

## 4. [TDD阶段] Repository 层单元测试

**Prompt**：
> 你是资深 Python 后端工程师。为 TranscriptRepository 编写单元测试，重点测试 `create` 方法的 `sequence` 自增逻辑：连续插入 3 条发言，验证 sequence 分别为 1、2、3。使用内存 SQLite 数据库（`:memory:`）和 pytest。

**意图**：用测试驱动开发，确保 `sequence` 生成逻辑正确且并发安全。

**挑战**：AI 最初未考虑并发插入时的 `sequence` 重复问题，仅用 `SELECT MAX(sequence)` 未加事务保护。

**修正**：要求使用 `SAVEPOINT` 包裹 `MAX(sequence)` 查询和 INSERT，确保原子性。

---

## 5. [TDD阶段] SSE 事件流集成测试

**Prompt**：
> 你是后端工程师。验证 SSE 事件流能否在讨论开始后推送 `speech`、`status_update`、`consensus_update`、`divergence_update` 事件。使用 curl 手动验证，确保事件格式与 OpenAPI 一致。

**意图**：确认实时推送机制正常工作，前端能正确接收并渲染事件。

**挑战**：SSE 长连接在 TestClient 中会阻塞，无法用标准 pytest 完成自动化测试。

**修正**：跳过自动化测试，改用 curl 手动验证，并在 `PROMPTS.md` 中记录该决策。

---

## 6. [E2E阶段] 端到端测试

**Prompt**：
> 你是测试工程师。模拟完整用户旅程：创建讨论 → 确认阵容 → 开始讨论 → 实时发言 → 结束讨论。使用 `httpx` 模拟 API 调用，验证每个步骤的响应状态和数据结构。

**意图**：验证系统端到端流程完整性，确保各模块协同工作。

**挑战**：前端 Mock 数据与真实 API 响应结构不一致（`speaker_name` / `speaker_color` 字段缺失）。

**修正**：统一前后端字段映射，在 Repository 层返回数据时补充冗余字段。

---

## 7. [优化阶段] 用户提问功能

**Prompt**：
> 你是全栈工程师。为系统增加“用户提问”功能：讨论室底部增加输入框，用户输入问题后，系统随机选择一位专家通过 DeepSeek API 生成回答，并通过 SSE 实时推送给所有观看者。后端新增 `/ask` 接口，前端新增提问组件。

**意图**：提升用户参与感，让讨论从“单向观看”变为“双向互动”。

**挑战**：专家回答仍然受历史发言影响，容易跑题。

**修正**：将 `_build_answer_prompt` 中的用户问题提到最前面，并明确要求“完全忽略讨论话题和历史发言”。（**后续优化方向：进一步强化 `generate_speech` 的聚焦能力**）
